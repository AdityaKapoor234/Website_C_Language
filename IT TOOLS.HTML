IT TOOLS & BUSINESS SYSTEMS

OPERATING SYSTEM

IT IS AN INTERFACE BETWEEN USER AND COMPUTER.

EXAMPLE-: WINDOWS, DOS, LINUX, UNIX, ETC.

FUNCTIONS-:

  1. PROCESSOR MANAGEMENT-:

    THE OPERATING SYSTEM ASIGNS PROCESSOR TO THE DIFFERENT TASKS THAT MUST BE PERFORMED BY THE COMPUTER.

  2. MEMORY MANAGEMENT-:

    IT ALLOCATES THE MAIN MEMORY AND SECONDARY MEMORY TO ALL THE PROGRAMMES AND DATA.

  3. INPUT/OUTPUT MANAGEMENT-:

    IT CARRIES OUT INPUT OUTPUT MANAGEMENT AND CORDINATES AND ASSIGNS DIFFERENT INPUT OUTPUT DEVICES.

  4. FILE MANAGEMENT-:

    IT MANAGES FILE ON VARIOUS STORAGE DEVICES AND TRANSFER THERE FILES FROM ONE DEVICE TO ANOTHER. IT ALLOWS THE FILES TO BE EASILY CHANGED AND MODIFIED BY USING DIFFERENT TEXT EDITOR.

  5. SCHEDULING-:

    OPERATING SYSTEM MAINTAINS THE ORDER IN WHICH PROCESS ARE TO BE EXECUTED IN THE COMPUTER SYSTEM.

  6. SECURITY MANAGEMENT-:

    IT MAINTAINS DATA SECURITY AND PROTECTS THE DATA FROM BEING DESTROYED BY ANY UNAUTHORISED USER.

TYPES OF OPERATING SYSTEM-:

  1. SINGLE TASKING OPERATING SYSTEM-:

    THE OPERATING SYSTEMS WHICH CAN EXECUTE ONLY SINGLE TASK AT A TIME.

    EXAMPLE-: DOS

  2. MULTI TASKING OPERATING SYSTEM-:

    THE OPERATING SYSTEMS WHICH SUPPORTS EXECUTION OF MORE THAN ONE TASK AT A TIME.

    EXAMPLE-: WINDOWS, LINUX, UNIX, ETC.

ANALOG COMPUTERS V/S DIGITAL COMPUTERS

ANALOG COMPUTERS DIGITAL COMPUTERS
1) ANALOG COMPUTERS HAVE LOW MEMORY. DIGITAL COMPUTERS HAVE A VERY LARGE MEMORY.
2) ANALOG COMPUTERS HAVE SLOW SPEED. DIGITAL COMPUTERS HAVE VERY FAST SPEED.
3) ANALOG COMPUTERS ARE LESS RELIABLE. DIGITAL COMPUTERS ARE MORE RELIABLE.
4) ANALOG COMPUTERS ARE USED IN ENGINEERING, SCIENCE AND MEDICAL FIELDS. DIGITAL COMPUTERS ARE USED IN ALL FIELDS OF LIFE.
5) EXAMPLE-: THERMOMETER, ANALOG CLOCK, SPEEDOMETER, ETC. EXAMPLE-: PERSONAL COMPUTERS, LAPTOPS, SMART PHONES, ETC.


INPUT DEVICES
  • KEYBOARD-:

    PROGRAMS AND DATA ARE ENTERED INTO A COMPUTER THROUGH A KEYBOARD WHICH IS ATTACHED TO A COMPUTER. A KEYBOARD IS SIMILAR TO THE KEYBOARD OF THE TYPEWRITER. IT CONTAINS LETTERS, DIGITS, SPECIAL CHARACTERS, FUNCTION KEYS AND SOME CONTROL KEYS. WHEN A KEY IS PRESSED, AN ELECTRONIC SIGNAL IS PRODUCED, WHICH IS DETECTED BY AN ELECTRONIC CIRCUIT CALLED KEYBOARD ENCODER.

  • MOUSE-:

    A MOUSE IS A POINTING DEVICE, WHICH IS HELD IN ONE HAND AND MOVED ACROSS A FLAT SURFACE. THE MOUSE CAN ALSO BE USED TO DRAW SKETCHES, DIAGRAM, ETC ON THE MONITOR SCREEN.

  • SCANNER-:

    A SCANNER IS A HARDWARE DEVICE WHICH IS USED TO SCAN TEXT OR IMAGES INTO THE COMPUTER. SOMETIMES THE INFORMATION, PICTURE OR TEXT THAT IS AVAILABLE ON PAPER IS NEEDED ON THE COMPUTER. THE BASIC FUNCTION OF A SCANNER IS TO CONVERT A DOCUMENT INTO DIGITAL FORMAT.

    THERE ARE 3 TYPES OF SCANNERS-:
    1.  FLATBED SCANNER
    2.  SHEETFED SCANNER
    3.  HANDHELD SCANNER


  • BAR CODE READER-:

    BAR CODE READERS ARE SPECIAL DEVICES USED TO READ BAR CODED DATA. BAR CODE IS A SPECIALIZED CODE USED FOR FAST IDENTIFICATION OF ITEMS. IT CONSISTS OF SERIES OF SMALL LINES, KNOWN AS BARS. THE ACTUAL CODING OF THE BARS IS THE WIDTH OF THE BAR, NOT THE HEIGHT. THESE ARE PRIMARILY USED FOR IDENTIFICATION OF GOODS, SUCH AS BOOKS, PACKAGES, POSTAL, ETC.

  • MAGNETIC INK CHARACHTER READER-:

    MICR DETCTS THE SPECIAL ENCODED CHARACTERS ON BANK CHEQUES AND DEPOSIT SLIPS. AFTER DETECTING THE ENCODED CHARACTERS, THE MICR CONVERTS THEM INTO DIGITAL DATA FOR THE COMPUTER.

OUTPUT DEVICES
  • MONITOR-:

    MONITOR ARE OF 2 TYPES-:
    1.  CRT (CATHODE RAY TUBE)
    2.  TFT-LCD (THIN FILM TRANSISTOR LIQUID CRYSTAL DISPLAY)


    • CRT (CATHODE RAY TUBE)

      A VDU (VISUAL DISPLAY UNIT) OR CRT MONITOR IS SIMILAR TO A TELEVISION AND ITS SIZE IS MEASURED IN DIAGONAL LENGTH OF THE SCREEN. IT SHOWS TEXT OR PICTURE IN COLOUR OR BLACK & WHITE, DEPENDING ON THE TYPE.

    • TFT-LCD (THIN FILM TRANSISTOR LIQUID CRYSTAL DISPLAY)

      LCD MONITOR IS THE FLAT PANEL TYPE OF MONITOR FOUND ON NOTEBOOK PCs. TFT-LCD IS A VARIANT OF LIQUID CRYSTAL DISPLAY WHICH USES THIN FLIM TRANSISTOR TECNOLOGY TO IMPROVE IMAGE QUALITY. TFT-LCD IS ONE TYPE OF ACTIVE MATRAX LCD. IT IS USED IN TELEVISION, FLAT PANEL DISPLAYS, PROJECTORS, ETC.

  • PRINTERS-:

    PRINTERS ARE THE MOST POPULAR OUTPUT DEVICES. THEY PROVIDE INFORMATION IN A PERMANENT READABLE FORM. THEY PRODUCE PRINTED OUTPUTS OF RESULTS, PROGRAMS AND DATA. A CHARACTER PRINTER PRINTS ONE CHARACTER OF THE TEXT AT A TIME. THEY ARE LOW-SPEED PRINTERS, WITH THIER SPEED RANGING FROM 300-400 CHARACTERS PER SECOND.

    CHARACTER PRINTERS CAN BE CLASSIFIED AS-:
    1.   IMPACT PRINTER
    2.   NON-IMPACT PRINTER
IMPACT PRINTER NON-IMPACT PRINTER
1) IT PRINTS CHARACTERS OR IMAGES BY STRIKING PRINT HAMMER OR WHEEL AGAINST AN INKED RIBBON. IT PRINTS CHARACTERS AND IMAGES WITHOUT STRIKING THE PAPERS.
2) ITS SPEED IS SLOWER. ITS SPEED IS FASTER.
3) ITS PRINTING QUALITY IS LOWER. ITS PRINTING QUALITY IS HIGHER.
4) IT NORMALLY USES CONTINUOUS PAPER SHEET. IT NORMALLY USES INDIVIDUAL PAPER SHEET.
5) EXAMPLE: DOT MATRIX. EXAMPLE: LASER PRINTER.
  • MULTIMEDIA PROJECTOR-:

    MULTIMEDIA PROJECTOR IS AN OUTPUT DEVICE CONNECTED TO A PC AND USED TO PROJECT INFORMATION FROM A COMPUTER ONTO A LARGE SCREEN (A CLOTH SCREEN OR WALL). THE INFORMATION IS THUS VIEWED BY A LARGE NUMBER OF PEOPLE. IT IS WIDELY USED FOR MAKING PRESENTATIONS.

CONVERSIONS
  1. DECIMAL TO BINARY-:

    DIVIDE THE DECIMAL NUMBER BY 2

    EXAMPLE-:



    (35)10=(100011)2

  2. BINARY TO DECIMAL-:

    MULTIPLY THE BINARY NUMBER BY 2N, ...... ,23,22,21,20

    EXAMPLE-:

    (100011)2=1 x 25 + 0 x 24 + 0x 23 + 0x 22 + 1 x 21 + 1 x 20

    (100011)2=(35)10

  3. DECIMAL TO OCTAL-:

    DIVIDE THE DECIMAL NUMBER BY 8

    EXAMPLE-:



    (67)10=(103)8

  4. OCTAL TO DECIMAL-:

    MULTIPLY THE BINARY NUMBER BY 8N, ...... ,83,82,81,80

    EXAMPLE-:

    (103)8=1 x 82 + 0 x 81 + 3 x 80

    (103)8=(67)10

  5. DECIMAL TO HEXADECIMAL-:

    DIVIDE THE DECIMAL NUMBER BY 16

    EXAMPLE-:



    (90)10=(5A)16

  6. HEXADECIMAL TO DECIMAL-:

    MULTIPLY THE BINARY NUMBER BY 16N, ...... ,163,162,161,160

    EXAMPLE-:

    (5A)16=5 x 161 + 10 x 160

    (5A)16=(90)10

  7. OCTAL TO BINARY-:

    OCTAL NUMBER'S BASE IS 8, i.e 8=23
    THEREFORE, OCTAL NUMBER IS OF 3 BITS

    THEREFORE, USE THE GIVEN TABLE FOR OCTAL TO BINARY CONVERSION.


    EXAMPLE-:  (34)8=(011 100)2

  8. BINARY TO OCTAL-:

    OCTAL NUMBER'S BASE IS 8, i.e 8=23
    THEREFORE, OCTAL NUMBER IS OF 3 BITS

    THEREFORE, USE THE GIVEN TABLE FOR BINARY TO OCTAL CONVERSION.


    EXAMPLE-:  (101 010)2=(52)8

  9. HEXADECIMAL TO BINARY-:

    HEXADECIMAL NUMBER'S BASE IS 16, i.e 16=24
    THEREFORE, HEXADECIMAL NUMBER IS OF 4 BITS

    THEREFORE, USE THE GIVEN TABLE FOR HEXADECIMAL TO BINARY CONVERSION.


    EXAMPLE-:  (C68)16=(1100 0110 1000)2

  10. BINARY TO HEXADECIMAL-:

    HEXADECIMAL NUMBER'S BASE IS 16, i.e 16=24
    THEREFORE, HEXADECIMAL NUMBER IS OF 4 BITS

    THEREFORE, USE THE GIVEN TABLE FOR BINARY TO HEXADECIMAL CONVERSION.


    EXAMPLE-:  (0010 1010)2=(2A)16

MEMORY
  • RAM(RANDOM ACCESS MEMORY)-:

    RAM IS A READ/WRITE MEMORY. INFORMATION CAN BE WRITTEN INTO AND READ FROM A RAM. IT IS A VOLATILE MEMORY. IT RETAINS THE STORED INFORMATION AS LONG AS IT IS SUPPLIED WITH POWER SUPPLY. WHEN POWER SUPPLY IS SWITCHED OFF OR INTERRUPTED, THE STORED INFORMATION IN THE RAM IS LOST.

  • ROM(READ ONLY MEMORY)-:

    ROM IS A PERMANENT TYPE MEMORY. ITS CONTENT ARE WRITTEN INTO AT MANUFACTURING TIME. ROMs STORE PERMANENT PROGRAMS AND OTHER TYPES OF DATA WHICH ARE NEEDED BY THE COMPUTER TO EXECUTE USER PROGRAMS. ROMs STORE FUNCTIONS SUCH AS SINE, COSINE, LOGARITHM, SQUARE ROOT, EXPONENTIAL AND CODE CONVERSION TABLES, ETC.

  • CACHE MEMORY-:

    CACHE MEMORY IS A MEMORY WHICH IS LOCATED BETWEEN PROCESSOR AND MAIN MEMORY. IT IS FASTER THAN MEMORY IT IS USED TO INCREASE THE OPERATING SPEED OF THE COMPUTER. IT IS COSTLIER THAN THE MAIN MEMORY AND SMALLER IN SIZE AS COMPARE TO MAIN MEMORY.

    IT IS A VOLATILE MEMORY. IT STORES THE INFORMATION TILL THE COMPUTER IS RUNNING OR IN TOUCH OF REGULAR POWER SUPPLY (CONSTANT SOURCE OF POWER SUPPLY) AND LOOSES ITS DATA/INFORMATION WHEN THE COMPUTER IS TURN OFF.

SYSTEM SOFTWARE V/S APPLICATION SOFTWARE

SYSTEM SOFTWARE APPLICATION SOFTWARE
1) IT GETS INSTALLED WHEN THE OPERATING SYSTEM IS INSTALLED ON COMPUTER. IT IS INSTALLED ACCORDING TO REQUIREMENT OF COMPUTER.
2) USER DO NOT INTERACT WITH SYSTEM SOFTWARE AS IT WORKS IN THE BACKGROUND. USER DIRECTLY INTERACT WITH IT WHILE DOING DIFFERENT ACTIVITIES.
3) USER MAY NOT REQUIRE MORE THAN ONE TYPE OF SYSTEM SOFTWARE. NUMBER OF APPLICATION SOFTWARE PROGRAMS INSTALLED ON THE COMPUTER AT A SAME TIME.
4) IT CAN RUN INDEPENDENTLY. IT CANNOT RUN WITHOUT THE PRESENCE OF THE SYSTEM SOFTWARE.
5) EXAMPLE: COMPILERS, DRIVERS, DEBUGGERS, ASSEMBLERS. EXAMPLE: MEDIAPLAYERS, WORD PROCESSORS, SPREADSHEET.


COMPILER V/S INTERPRETER

COMPILER INTERPRETER
1) COMPILER TAKES ENTIRE PROGRAM AS INPUT. INTERPRETER TAKES SINGLE INSTRUCTION AS INPUT.
2) INTERMEDIATE OBJECT CODE IS GENERATED. NO INTERMEDIATE OBJECT CODE IS GENERATED.
3) CONDITIONAL CONTROL STATEMENTS ARE EXECUTED FASTER. CONDITIONAL CONTROL STATEMENTS ARE EXECUTED SLOWER.
4) MEMORY REQUIREMENT : MORE (SINCE OBJECT CODE IS GENERATED). MEMORY REQUIREMENT IS LESS.
5) PROGRAM NEED NOT BE COMPILED EVERY TIME. EVERY TIME HIGHER LEVEL PROGRAM IS CONVERTED INTO LOWER LEVEL PROGRAM.
6) ERRORS ARE DISPLAYED AFTER ENTIRE PROGRAM IS CHECKED. ERRORS ARE DISPLAYED FOR EVERY INSTRUCTION INTERPRETED (IF ANY).
7) EXAMPLE : C Compiler EXAMPLE : BASIC
